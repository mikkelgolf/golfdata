## Task
we need to make a small correction to the NCAA championship records and revisit the NCAA "Champions By Year" section. For 1940, Princeton and LSU both finished T1, making them both winners. For 1942, LSU and Stanford are both T1, making them both winners. That needs to be reflected in both the Champions By Year table, so both teams should appear in that yearly box, and both should have credit in the table below that AND their individual team pages

## Actions

Investigated the Champions By Year rendering pipeline and found two
independent bugs preventing tied-for-first co-champions from showing up:

1. **Champion predicate too narrow.** `isChampion()` in `src/lib/streaks.ts`
   only accepted `position === "1"` for the pre-2009 stroke-play era,
   silently dropping any `"T1"` row. 1942's LSU/Stanford co-championship
   was therefore not credited to either program in the Champions By Year
   grid, the team-totals table, or the individual team pages. Widened
   the predicate to `"1" || "T1"`; mirrored the matching predicate in
   `scripts/build-championships-history.ts`. Match-play-era logic
   (`wonChampionship === true`) is unchanged.

2. **1940 data error in source CSV.** The `/tmp/ncaa_sheet_dump/MEN_-_NCAA_Final_Results.csv`
   dump listed LSU as outright 1940 champion with Princeton as solo
   runner-up, but they actually tied T1. David updated the upstream
   Google sheet manually. Added a post-parse `CORRECTIONS` layer in the
   build script that rewrites `(gender, year, team)` rows so the fix is
   idempotent across rebuilds and doesn't depend on the `/tmp` dump
   being refreshed.

Regenerated `src/data/championships-history.json` via
`npx tsx scripts/build-championships-history.ts`. TypeScript + ESLint
clean.

## Open questions / learnings

- The `CORRECTIONS` layer is a lightweight extension point for future
  upstream CSV errors. Worth reusing rather than editing the JSON
  directly.
- The comment block above `buildWinnersByYear` in
  `src/components/championships-history-table.tsx` (lines 172–176)
  already claimed "Pre-2009 ties produce multiple champions (e.g., 2015
  men's: LSU & Stanford)" — but `isChampion` broke that contract for any
  year with real T1 finishes. The 2015 example in the comment is a
  match-play-era year so it worked; the pre-match-play T1s silently
  didn't. Worth auditing the full pre-2009 range for any other T1 rows
  that may still be dropped or miscoded as outright `"1"` in the CSV.
- The `/tmp/ncaa_sheet_dump/` path is machine-local and ephemeral.
  Long-term we should consider checking a versioned copy into the repo
  or a more formal import pipeline.
